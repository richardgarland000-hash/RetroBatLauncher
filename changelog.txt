# Changelog

All notable changes to this project will be documented in this file.

The format is based on Keep a Changelog,
and this project adheres to Semantic Versioning.

---

## [2.5.2] - 2026-04-24

### Changed
- Added download link buttons for version failure message of DirectX, 
  Visual C++ 2010 and Visual C++ 2015-2022 in results window. 

---

## [2.5.1] - 2026-04-24

### Changed
- Modified launcher.py to make version displayed in splash window either
  by assigning version in self.version_text or by version in version_info.txt
  with the benefit of the compiled executable being completely self-contained
  as opposed to needing both launcher.py and version_info.txt file updated and
  included in the deployment.

---

## [2.5.0] - 2026-04-23

### Changed
- Ditched immediate splash exit and message box when RetroBat exe location is
  not found (step 1 of do_step) in favor of adding to the other checks verified
  in show_results_window.
- Changes to show_results_window(), step 1 of do_step(i) and behavior after splash
  closes. Results window remains open with an exit button per any verification
  failure or immediately closes and triggers callback to launch Retrobat.
  Previous start button removed in favor of the autostart.

---

## [2.0.0] - 2026-04-22

Major rewrite to fix each verification and accumulate the results in an
array. Other than if retrobat.exe could not be found, all verifications 
are completed and the splash screen closes with a results window opening.
A start button or an exit button is at the bottom of the results window 
depending on pass/fail of all verifications. 

Expanded information logged each run for diagnostics. 

Moved each verification script to external files to reduce number of lines 
in launcher.py.
