## Code of Conduct
By participating in this project, you agree to abide by our [Code of Conduct](https://github.com/richardgarland000-hash/RetroBatLauncher/blob/master/CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs
Bugs are tracked as [GitHub issues](https://github.com/richardgarland000-hash/RetroBatLauncher/issues). When filing a bug, please include:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Expected behavior vs. actual results.
- Your environment details (OS, version, etc.).

### Suggesting Enhancements
Enhancement suggestions are also tracked as issues. Please describe the current behavior and the specific change you'd like to see, explaining why it would be useful.

### Your First Code Contribution
1. Fork the repository.
2. Create a topic branch from `master`: `git checkout -b feature/your-feature-name`.
3. Make your changes and commit them with descriptive [commit messages](https://chris.beams.io/posts/git-commit/).
4. Push your branch to your fork.

## Pull Request Process
1. Ensure all tests pass and your code follows our style guide.
2. Open a Pull Request (PR) against the `master` branch.
3. Link the PR to any related issues.
4. A maintainer will review your PR and provide feedback.

## Style Guides
- **Git Commits:** Use the imperative mood (e.g., "Add feature" instead of "Added feature").
- **Code Style:** Please follow the existing [Style Guide](https://peps.python.org/pep-0008/) used in this project.
